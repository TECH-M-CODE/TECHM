# SentinelAI — Canonical Domain Model

This document defines the frozen canonical domain model for SentinelAI. These entities act as the ultimate source of truth across the database, repositories, and API layer.

## 1. Company
Represents a corporate entity being monitored.
- **Fields**: 
  - `id` (UUID, PK)
  - `name` (String, Indexed)
  - `registration_number` (String, Optional)
  - `jurisdiction` (String)
  - `sector` (String, Optional)
  - `risk_score` (Float, Default: 0.0)
  - `risk_band` (Enum: LOW, MEDIUM, HIGH, CRITICAL)
  - `status` (Enum: ACTIVE, INACTIVE, UNDER_INVESTIGATION)
  - `created_at` (DateTime)
  - `updated_at` (DateTime)
- **Relationships**: 
  - `has_many` Person (Directors/UBOs) via `EntityPersonLink`
  - `has_many` RiskEvent
  - `has_many` Alert
  - `has_many` SARReport
- **Lifecycle**: Created during onboarding -> Monitored continuously -> Risk Score updated via Events -> May trigger Alerts/Investigations.
- **Indexes**: `idx_company_name`, `idx_company_risk_band`

## 2. Person
Represents an individual (e.g., UBO, Director, PEP).
- **Fields**:
  - `id` (UUID, PK)
  - `full_name` (String, Indexed)
  - `date_of_birth` (Date, Optional)
  - `nationality` (String, Optional)
  - `is_pep` (Boolean, Default: False)
  - `risk_score` (Float, Default: 0.0)
  - `risk_band` (Enum: LOW, MEDIUM, HIGH, CRITICAL)
- **Relationships**:
  - `belongs_to_many` Company via `EntityPersonLink`
  - `has_many` RiskEvent
- **Lifecycle**: Created via corporate linkage or direct onboarding -> Screened against watchlists -> Updates propagate to linked Companies.
- **Indexes**: `idx_person_name`

## 3. EntityPersonLink
Association table for Company <-> Person.
- **Fields**:
  - `id` (UUID, PK)
  - `company_id` (UUID, FK -> Company.id)
  - `person_id` (UUID, FK -> Person.id)
  - `role` (Enum: UBO, DIRECTOR, SHAREHOLDER, SIGNATORY)
  - `ownership_percentage` (Float, Optional)
- **Indexes**: `idx_link_company`, `idx_link_person`

## 4. NewsEvent
Raw, unprocessed signals from external sources.
- **Fields**:
  - `id` (UUID, PK)
  - `content_hash` (String, UK) - SHA256 of content for deduplication
  - `source_url` (String)
  - `title` (String)
  - `content` (Text)
  - `published_at` (DateTime)
  - `credibility_score` (Float, populated by Verification Engine)
  - `status` (Enum: PENDING, PROCESSED, DISMISSED, FAILED)
- **Relationships**:
  - `has_many` RiskEvent (if applicable to an entity)
- **Lifecycle**: Ingested -> Deduped -> Verification -> Screening -> Generates RiskEvent(s) -> Marked Processed.

## 5. WatchlistEntry
Cached copy of external sanctions/PEP lists (OFAC, UN, EU).
- **Fields**:
  - `id` (UUID, PK)
  - `external_id` (String, Indexed)
  - `list_source` (Enum: OFAC, UN, EU, UK_HMT)
  - `entity_type` (Enum: COMPANY, PERSON, VESSEL)
  - `primary_name` (String, Indexed)
  - `aliases` (JSON)
  - `metadata` (JSON - DOBs, nationalities, addresses)
  - `is_active` (Boolean, Default: True)
  - `version` (Integer)
- **Lifecycle**: Upserted during scheduled syncs -> Used for rapid fuzzy screening.

## 6. ScreeningMatch
Record of a fuzzy/LLM match between a NewsEvent/Entity and a WatchlistEntry.
- **Fields**:
  - `id` (UUID, PK)
  - `entity_id` (UUID, FK -> Company.id or Person.id)
  - `watchlist_id` (UUID, FK -> WatchlistEntry.id)
  - `match_type` (Enum: EXACT, FUZZY, SEMANTIC)
  - `confidence_score` (Float)
  - `llm_reasoning` (Text)
  - `status` (Enum: TRUE_POSITIVE, FALSE_POSITIVE, PENDING_REVIEW)
- **Relationships**: Links Entity to WatchlistEntry.

## 7. RiskEvent
A processed, scored event tied to a specific entity.
- **Fields**:
  - `id` (UUID, PK)
  - `entity_id` (UUID, FK -> Company.id or Person.id)
  - `source_event_id` (UUID, FK -> NewsEvent.id, Optional)
  - `event_category` (Enum: SANCTION, ADVERSE_MEDIA, PEP_UPDATE, TRANSACTION)
  - `severity` (Enum: LOW, MEDIUM, HIGH, CRITICAL)
  - `score_delta` (Float)
  - `reasoning` (Text)
  - `created_at` (DateTime)
- **Lifecycle**: Generated by Scoring Engine -> Updates Entity Risk Score -> May trigger Alert -> Rendered in Decision Graph.

## 8. Alert
Requires human intervention due to high risk or velocity.
- **Fields**:
  - `id` (UUID, PK)
  - `entity_id` (UUID, FK)
  - `trigger_event_id` (UUID, FK -> RiskEvent.id)
  - `priority` (Enum: LOW, MEDIUM, HIGH, CRITICAL)
  - `status` (Enum: OPEN, IN_PROGRESS, ESCALATED, RESOLVED, DISMISSED)
  - `assigned_to` (UUID, FK -> User.id, Optional)
  - `created_at` (DateTime)
  - `resolved_at` (DateTime, Optional)
- **Relationships**:
  - `has_one` Investigation
  - `has_one` SARReport

## 9. Investigation
Compiled evidence bundle for an Alert.
- **Fields**:
  - `id` (UUID, PK)
  - `alert_id` (UUID, FK -> Alert.id)
  - `summary` (Text)
  - `evidence_bundle` (JSON - list of cited sources, links, and reasoning)
  - `decision_trace` (JSON - graph structure for UI rendering)
  - `created_at` (DateTime)

## 10. SARReport (Suspicious Activity Report)
Draft narrative for regulatory filing.
- **Fields**:
  - `id` (UUID, PK)
  - `alert_id` (UUID, FK)
  - `entity_id` (UUID, FK)
  - `version` (Integer)
  - `narrative` (Text)
  - `citations` (JSON - regulatory references)
  - `status` (Enum: DRAFT, PENDING_APPROVAL, APPROVED, REJECTED, FILED)
  - `created_by` (UUID, FK -> User.id)
  - `approved_by` (UUID, FK -> User.id, Optional)
- **Lifecycle**: Drafted by Agent -> Reviewed by Human -> Updated (creates new version) -> Approved -> Filed.

## 11. AuditLog
Tamper-evident, hash-chained ledger of ALL actions.
- **Fields**:
  - `id` (UUID, PK)
  - `entity_id` (UUID, FK, Optional)
  - `actor_id` (UUID, FK -> User.id or 'SYSTEM')
  - `action` (String - e.g., 'SCORE_UPDATED', 'SAR_APPROVED', 'ALERT_DISMISSED')
  - `payload` (JSON - snapshot of data changed)
  - `previous_hash` (String)
  - `entry_hash` (String, UK) - SHA256(previous_hash + payload + timestamp)
  - `signature` (String, Optional)
  - `timestamp` (DateTime)
- **Lifecycle**: Append-only. Never updated. Never deleted.

## 12. User
System operator (Analyst, Manager, Admin).
- **Fields**:
  - `id` (UUID, PK)
  - `email` (String, UK)
  - `hashed_password` (String)
  - `role` (Enum: L1_ANALYST, L2_INVESTIGATOR, MLRO, ADMIN)
  - `is_active` (Boolean)

## 13. TimelineEvent (UI Projection Model)
Not a DB table, but a strictly typed DTO for the Decision Graph UI.
- **Fields**:
  - `id` (String)
  - `type` (Enum: NEWS, MATCH, SCORE_CHANGE, ALERT, HUMAN_ACTION)
  - `timestamp` (DateTime)
  - `title` (String)
  - `description` (String)
  - `metadata` (JSON)
  - `edges` (List[String] - IDs of related events to draw the React Flow graph)
